# 🏛️ Museum Viewer PRO

Интерактивная программа для просмотра 3D-моделей музейных экспонатов с описаниями и режимом авто-показа.

---

## 📁 Структура проекта

```
museum_viewer/
│
├── museum_viewer.exe          ← Главный исполняемый файл
├── museum_viewer.py           ← Исходный код (если запуск из Python)
│
├── models/                    ← Папка с 3D-моделями
│   ├── 01_Виниловый проигрыватель.glb
│   ├── 02_Керосиновая лампа.glb
│   └── 03_Противотанковая винтовка.glb
│
└── text/                      ← Папка с текстовыми описаниями
    ├── Виниловый проигрыватель.txt
    ├── Керосиновая лампа.txt
    └── Противотанковая винтовка.txt
```

> ⚠️ Папки `models/` и `text/` должны находиться **в той же папке**, что и `museum_viewer.exe`.
> Если папок нет — программа создаст `models/` автоматически при первом запуске. Папку `text/` нужно создать вручную.

## 🚀 Запуск программы

### Из exe-файла
Просто дважды щёлкните `museum_viewer.exe`. Никакой установки не требуется.

### Из исходного кода (Python)
```bash
# Установка зависимостей
pip install PyQt5 panda3d

# Запуск
python museum_viewer.py
```

**Требования:** Python 3.10+, PyQt5, Panda3D.

## 🗂️ Как добавить новый экспонат

### Шаг 1 — Подготовьте 3D-модель

Программа поддерживает только формат **`.glb`** (бинарный glTF). Это современный стандарт 3D для веб и приложений.

Если модель у вас в другом формате — конвертируйте:

| Из чего | Как конвертировать |
|---|---|
| `.obj`, `.fbx`, `.dae` | Blender → File → Export → glTF 2.0 (.glb) |
| `.stl` | Blender или онлайн-конвертер (например, products.aspose.com) |
| `.3ds`, `.max` | Blender (импорт через плагин) → экспорт в .glb |

### Шаг 2 — Назовите файл по правилу

Имя файла определяет **порядок отображения** в списке и **название экспоната**:

```
NN_Название экспоната.glb
```

- `NN` — двузначный (или более) номер для сортировки: `01`, `02`, `10`, `99` и т.д.
- `_` — разделитель (можно также использовать пробел или точку)
- `Название экспоната` — то, что будет показано в списке и в заголовке описания

**Примеры правильных имён:**
```
01_Виниловый проигрыватель.glb   → в списке: "Виниловый проигрыватель"
02_Керосиновая лампа.glb          → в списке: "Керосиновая лампа"
10_Самовар XIX века.glb           → в списке: "Самовар XIX века"
```

Если номера нет — файл всё равно появится в списке, но окажется в самом конце.

### Шаг 3 — Поместите модель в папку `models/`

```
museum_viewer/
└── models/
    └── 04_Ваш экспонат.glb   ← сюда
```

### Шаг 4 — Создайте описание (необязательно, но рекомендуется)

Создайте текстовый файл `.txt` в папке `text/`. Имя файла должно **точно совпадать** с частью после номера в имени модели (без расширения `.glb`):

| Файл модели | Файл описания |
|---|---|
| `04_Самовар XIX века.glb` | `text/Самовар XIX века.txt` |
| `01_Виниловый проигрыватель.glb` | `text/Виниловый проигрыватель.txt` |

**Содержимое файла** — обычный текст в кодировке **UTF-8**. Можно писать несколько абзацев через пустую строку. Рекомендуемая длина — до 500 символов, чтобы описание помещалось без прокрутки.

**Пример `Самовар XIX века.txt`:**
```
Самовар XIX века, Тула, около 1860 года. Латунь, гравировка.

Использовался в купеческих домах для приготовления чая. Нагрев
осуществлялся углями, засыпаемыми в центральную трубу.
Украшен растительным орнаментом в стиле русского классицизма.

Высота: 54 см. Объём: 6 литров.
Поступил в коллекцию в 1923 году.
```

> Если файл `.txt` не найден — при нажатии кнопки «Описание» программа покажет подсказку с точным именем файла, который нужно создать.

## 🎮 Управление интерфейсом

### Левая панель

| Элемент | Описание |
|---|---|
| Список экспонатов | Клик по названию — переключает модель |
| Кнопка «📋 Описание» | Открывает / скрывает окно с описанием текущего экспоната |
| Подпись внизу | Показывает имя текущего экспоната |

### Верхняя панель

| Элемент | Описание |
|---|---|
| ☑ Режим мыши | Включает/выключает вращение модели мышью |
| ▶ Авто-показ | Запускает автоматическую демонстрацию (смена через 10 сек.) |
| ⏱ N сек. | Таймер до смены следующего экспоната |

### Кнопки навигации

| Кнопка | Действие |
|---|---|
| ⬅ Предыдущий экспонат | Переключить на предыдущий в списке |
| Следующий экспонат ➡ | Переключить на следующий в списке |
| ＋ Приблизить | Зум вперёд |
| － Отдалить | Зум назад |
| ↺ Сброс зума | Вернуть масштаб по умолчанию |

### Панель «Позиция» (правый край)

Стрелки позволяют смещать модель по горизонтали и вертикали (панорамирование). Кнопка ⊙ в центре сбрасывает смещение.

## 📋 Окно описания

При нажатии кнопки «📋 Описание экспоната» открывается плавающее окошко:

- **Заголовок** — название текущего экспоната
- **Текст** — содержимое соответствующего `.txt`-файла
- **Кнопка ✕** — закрыть окно
- Окно можно **перетаскивать** в любое место экрана, удерживая ЛКМ
- Если текст длинный — появляется вертикальная прокрутка
- При смене экспоната (вручную или через авто-показ) — **текст обновляется автоматически**, если окно открыто

## ▶️ Режим авто-показа

Нажмите «▶ Авто-показ» для запуска слайд-шоу:

1. Модель начинает **медленно вращаться**
2. **Автоматически открывается описание** текущего экспоната
3. Через **10 секунд** — переключение на следующий экспонат
4. Описание обновляется вместе со сменой модели
5. После последнего экспоната список начинается сначала (зацикливание)

Нажмите «⏹ Остановить» для выхода из режима авто-показа.

> Если в режиме авто-показа кликнуть по экспонату в списке — авто-показ останавливается и происходит ручное переключение.

### Архитектура: как PyQt5 и Panda3D работают вместе

Главная сложность проекта — интеграция двух независимых UI-фреймворков. Panda3D обычно сам создаёт своё окно, но здесь его 3D-рендер встраивается **внутрь виджета PyQt5**:

```
┌─────────────── QMainWindow (PyQt5) ─────────────────┐
│  ┌──────────────┐   ┌────────────────────────────┐  │
│  │  Left Panel  │   │    ViewWidget (QWidget)     │  │
│  │  (QListWidget│   │                            │  │
│  │   + buttons) │   │  ← Panda3D рендерит сюда  │  │
│  └──────────────┘   │    через winId()           │  │
│                     └────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

Ключевой момент: `engine.attach(self.view.winId())` — передаёт системный идентификатор окна PyQt5 в Panda3D, и движок открывает своё окно рендера **внутри** Qt-виджета.

---

### Классы и их роли

#### `Engine(ShowBase)` — 3D-движок
Наследует `panda3d.direct.showbase.ShowBase`. Отвечает за всё, что происходит в 3D:

```
Engine
├── attach(win_id)       — встраивает рендер в Qt-виджет
├── load_model(path)     — загружает .glb, центрирует и масштабирует
├── do_zoom(delta)       — изменяет дистанцию камеры
├── do_pan(dx, dz)       — смещает камеру по X и Z
├── mouse_down/up()      — фиксирует начало/конец перетаскивания
├── update_mouse()       — пересчитывает вращение (с инерцией)
└── auto_rotate(speed)   — вращение для авто-показа
```

**Камера:** Не перемещается вокруг модели — вместо этого модель вращается сама (через `setH`, `setP`). Камера смотрит на `(pan_x, 0, pan_z)` с позиции `(pan_x, zoom, pan_z)`.

**Инерция вращения:** При отпускании мыши скорость не обнуляется мгновенно, а затухает с коэффициентом `inertia_strength = 0.92` за каждый тик.

#### `ViewWidget(QWidget)` — область просмотра
Тонкая обёртка над `QWidget`. Пробрасывает:
- `wheelEvent` → зум через `Engine.do_zoom()`
- `resizeEvent` → `Engine.resize_window()` чтобы Panda3D знала о новом размере

#### `DescriptionDialog(QWidget)` — окно описания
Отдельный плавающий виджет (`Qt.Tool | Qt.FramelessWindowHint`):
- Создаётся один раз при старте и переиспользуется (просто скрывается/показывается)
- `set_content(title, text)` обновляет содержимое и вызывает `_adjust_size()` для подстройки высоты
- Поддерживает перетаскивание через `mousePressEvent` / `mouseMoveEvent`
- Максимальная высота текстовой зоны — 450px, выше появляется прокрутка

#### `Window(QMainWindow)` — главное окно
Связывает всё вместе:
- Строит весь UI в методе `ui()`
- `load_models()` — сканирует `models/*.glb`, парсит имена через регулярное выражение `(\d+)[_. ]*(.+)`
- `show_model(i)` — загружает модель и обновляет описание (если открыто)
- `tick()` — вызывается каждые 16 мс через `QTimer`. Делает один шаг Panda3D-задач (`taskMgr.step()`) и обновляет мышь/слайдшоу

---

### Главный цикл (тик)

```
QTimer (16 мс) → Window.tick()
                    ├── engine.taskMgr.step()       ← шаг Panda3D (рендер, события)
                    └── если slideshow_active:
                            slideshow_tick()
                                ├── engine.auto_rotate()
                                ├── обновить таймер на экране
                                └── если время вышло → show_model(next) + _open_description()
                        иначе:
                            engine.update_mouse()    ← вращение по мыши с инерцией
```

16 мс ≈ 60 кадров в секунду.

### Загрузка и масштабирование модели

```python
def load_model(self, path):
    m = loader.loadModel(path)
    bounds = m.getTightBounds()       # получаем AABB (ось-выравненный ящик)
    center = (min + max) * 0.5        # центр модели
    m.setPos(-center)                 # сдвигаем модель так, чтобы центр был в (0,0,0)
    size = (max - min).length()       # диагональ ящика
    m.setScale(2.5 / max(size, 1))    # масштабируем до единичного размера
```

Это гарантирует, что любая модель — маленькая монета или большая пушка — будет видна в одном и том же масштабе.

```
load_description(name)
```

```python
if self.desc_dialog.isVisible():

    self.desc_dialog.set_content(name, text)
```

### Путь к файлам (exe vs скрипт)

```python
def get_models_path():
    if getattr(sys, 'frozen', False):   # запущено как .exe (PyInstaller)
        base_path = os.path.dirname(sys.executable)
    else:                               # запущено как .py
        base_path = os.path.dirname(os.path.abspath(__file__))
    return Path(base_path) / "models"
```

`sys.frozen` устанавливается PyInstaller при упаковке в exe. Это позволяет одному коду работать и как скрипт, и как exe-файл.

## ❓ Частые вопросы

**Модель не появляется в списке?**
Убедитесь, что файл имеет расширение `.glb` (не `.gltf`) и находится прямо в папке `models/`, не в подпапке.

**Модель загружается, но выглядит чёрной или без текстур?**
GLB-файл должен содержать текстуры внутри себя (embedded). При экспорте из Blender выбирайте «Format: glTF Binary (.glb)» — текстуры упаковываются автоматически.

**Кнопка «Описание» показывает «файл не найден»?**
Создайте папку `text/` рядом с `.exe` и добавьте файл с именем точно как в подсказке (с учётом регистра и пробелов).

**Программа зависает при запуске?**
Это может происходить при первой инициализации Panda3D (создание контекста OpenGL). Подождите 3–5 секунд. Если не помогает — убедитесь, что драйверы видеокарты обновлены.

**Как изменить время авто-показа?**
В исходном коде найдите строку:
```python
self.SLIDESHOW_DURATION = 10000  # миллисекунды
```
Замените `10000` на нужное значение (например, `30000` = 30 секунд).
